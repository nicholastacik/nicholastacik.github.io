## Scraping J-Archive

There's no API, so the data comes from a polite, cached crawl of
[J-Archive](https://j-archive.com): a `fetch` stage (one request/second, every page cached
to disk), a `parse` stage that reads each game's board out of the HTML, and a resumable
`crawl` that walks every season and game. The raw crawl stays on my machine; what ships is
a single compact `clues.parquet` — one row per clue — so anyone (including the CI that
builds this site) starts from identical data without re-scraping.

The result is **563,266 clues** across **9,485 games**, spanning 1984 to 2026.

## From categories to documents

To compare categories, each one becomes a short text "document": its title followed by its
clue → answer pairs (in a shuffled order, so no clue position carries special weight). For
example, one category instance reads:

```{python}
#| label: sample-doc
#| echo: false
inst = clusters.merge(
    clues, on=["game_id", "round", "category"]
).groupby(["game_id", "round", "category"]).first().reset_index()
row = clues[clues["category"] == "FISH, BIRD OR MAMMAL"].head(4)
print("FISH, BIRD OR MAMMAL. " + "; ".join(f"{c} -> {a}" for c, a in zip(row["clue"], row["answer"])))
```

Each document is embedded into a 384-dimensional vector with a local sentence-transformer
(`bge-small-en-v1.5`) — a step heavy enough (a GPU-less few minutes over 123k documents)
that it runs offline and commits its *output*, never re-running when the post builds.

## Clustering into "types"

Running K-means (k = 50) over those vectors groups the ~123k category instances into
recognizable *types*. Projected to 2D with UMAP, the map looks like this — hover any point
to see its type, an example category, and that type's most common entity:

```{python}
#| label: fig-clusters
#| fig-cap: "123k category instances, grouped into 50 types (hover to explore; ~10k-point sample)."
frac = min(1.0, 10000 / len(clusters))
samp = clusters.groupby("cluster_id", group_keys=False).apply(
    lambda g: g.sample(frac=frac, random_state=42)
)
samp["type"] = samp["cluster_id"].map(labels)
top1 = tokens[tokens["rank"] == 1].set_index("cluster_id")["phrase"].to_dict()
samp["top_entity"] = samp["cluster_id"].map(top1).fillna("—")

fig = px.scatter(
    samp, x="umap_x", y="umap_y", color="type",
    hover_name="type",
    hover_data={"category": True, "top_entity": True,
                "umap_x": False, "umap_y": False, "type": False, "cluster_id": False},
    render_mode="webgl", opacity=0.55,
)
fig.update_traces(marker={"size": 4})
fig.update_layout(showlegend=False, height=620,
                  margin={"l": 0, "r": 0, "t": 10, "b": 0},
                  xaxis_title=None, yaxis_title=None)
fig.show()
```

```markdown
## What the data says to study

Two findings stand out. First, **wordplay is the single biggest meta-domain** — homophones,
anagrams, rhymes, hidden words, "___-letter words." Grouped together, roughly one category
in ten rewards a *skill* you drill rather than facts you memorize. Push the clustering finer
and wordplay even splits into two prep tracks: pure letter/anagram games (drill) versus
etymology and idioms (genuinely studyable).

Second, the types differ enormously in whether "study the common answers" even *works*. We
can measure it: how many distinct entities recur ≥5 times in a type. Entity-dense types
(authors, geography, presidents) are eminently studyable; wordplay and grab-bag types are
not:

```{python}
#| label: fig-applicability
#| fig-cap: "How studyable each type is: distinct entities recurring 5+ times."
ap = (tokens.groupby("cluster_id")["n_qualifying_phrases"].first().reset_index())
ap["type"] = ap["cluster_id"].map(labels)
ap = ap.sort_values("n_qualifying_phrases")
top = ap.tail(12); bottom = ap.head(8)
show = pd.concat([bottom, top])
plt.figure(figsize=(8, 9))
colors = ["#C44E52"] * len(bottom) + ["#55A868"] * len(top)
plt.barh(show["type"], show["n_qualifying_phrases"], color=colors)
plt.xlabel("distinct recurring entities (count >= 5)")
plt.tight_layout()
plt.show()
```

And for the studyable types, here are the entities that actually recur — the stuff worth
knowing:

```{python}
#| label: tbl-entities
#| echo: false
def top_entities(name, n=10):
    cid = next(c for c, nm in labels.items() if nm == name)
    d = tokens[(tokens["cluster_id"] == cid) & (tokens["phrase"].notna())].head(n)
    return pd.DataFrame({name: d["phrase"].values, "count": d["count"].values})

from IPython.display import display
for name in ["U.S. Presidents", "World Geography & Waters", "Books & Authors"]:
    display(top_entities(name))
```
